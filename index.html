Qubob

Qubob is example usage of django-bob library for Django https://bitbucket.org/thesheep/bob/

How to run?

Virtualenv (optional)

python -m virtualenv --no-site-packages ~/qubob
cd qubob/

Required:

git clone https://github.com/quamilek/qubob.git
source bin/activate
cd qubob/
pip install -e .
cd src
./manage.py runserver

And open this link in browser: http://127.0.0.1:8000

DataTableMixin usage

Your view class must inherited DataTableMixin from bob.data_table. Define fields in class sort_variable_name, row_per_pages, export_variable_name and diefine columns - is a list of DataTableColumn class object contains fields needed for the generation of table header.

DataTableColumn fields

  • header_name - name to display in columns header
  • field - field name in model
  • type - icon type generated by bob header_table tag
  • selectable - header in bob tag is selectable, if you use this option you must include bootstrap/js/bob.js file in your template
  • bob_tag - set if the column is to be generated by bob tag
  • sort_expression - example book__size (book.size)
  • export - set when the column is to be exported

Data nedded to generate Data table must be return to the template in get_context_data. just that, copy this code:

def get_context_data(self, *args, **kwargs):
        ret = super(DataTableView, self).get_context_data(*args, **kwargs)
        ret.update(
            super(DataTableView, self).get_context_data_paginator(
                *args,
                **kwargs
            )
        )
        ret.update({
            'sort_variable_name': self.sort_variable_name,
            'url_query': self.request.GET,
            'sort': self.sort,
            'columns': self.columns,
        })
        return ret

Override get method

Just that, copy this cod, and replace ExampleClass to name your view class:

def get(self, *args, **kwargs):
        self.get_query()
        if self.export_requested():
            return self.response
        return super(ExampleClass, self).get(*args, **kwargs)

data_table_query

You must assign to self.data_table_query your queryset, preferably at the beginning of get method example:

def get(self, *args, **kwargs):
    books = Book.objects.all()
    self.data_table_query(books)
    if self.export_requested():
        return self.response
    return super(ExampleClass, self).get(*args, **kwargs)

Export CSV file

If you want to export data to CSV file you must ovverride get_csv_data method. This method must be return nestead list contains string data. Example:

[
    ['Programming Python', 'Feb. 12, 2013, 11:35 a.m.', '37.11',],
    ['Python for Data Analysis', 'Feb. 12, 2013, 11:36 a.m.', '22.31',],
]

Method get_csv_header from DataTableMixin return a dict with columns names defined in self.columns. Can be used as a first line of CSV file.

Template

In template must add a two tag from BOB.

{% table_header columns url_query sort fugue_icons %}

and

{% pagination bob_page url_query=url_query show_all=0 show_csv=1 fugue_icons=0 %}

First display a heder of table, the second is responsible for pagination. You should show the data from the query. Example tamplate:

<table class="table table-bordered table-striped table-properties">
    <tr>
        {% table_header columns url_query sort fugue_icons %}
    </tr>
    {% for f in bob_page %}
        <tr>
            <td class="column-checkbox">
                <input type="checkbox" name="select" value="{{ f.id }}">
            </td>
            <td>{{ f.title}}</td>
            <td>{{ f.pub_date }}</td>
            <td>{{ f.price }}</td>
        </tr>
    {% endfor %}
</table>
{% pagination bob_page url_query=url_query show_all=0 show_csv=1 fugue_icons=0 %}
